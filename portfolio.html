Caching Anti-Patterns

Caching infrequently accessed data leads to memory bloat and eviction of high-value keys. Use the 80/20 rule.

Stale data is worse than no data. Always set TTLs, even if long, to ensure eventual consistency.

Eviction Policy Guide

allkeys-lru

Best for general purpose caching. Keeps frequently used keys.

volatile-ttl

Evicts keys with an expire set and a shorter TTL first.

noeviction

Returns errors when memory limit is reached. Best for critical DB usage.
